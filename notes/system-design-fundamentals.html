Scalability Concepts

Horizontal vs Vertical Scaling

Horizontal scaling (scaling out) involves adding more servers to handle increased load, while vertical scaling (scaling up) means upgrading existing hardware.

Load Balancing

Distributes incoming requests across multiple servers. Key types:

  • Round Robin: Requests distributed sequentially
  • Least Connections: Routes to server with fewest active connections
  • IP Hash: Uses client IP to determine server

Database Design Patterns

ACID Properties

  • Atomicity: All or nothing transactions
  • Consistency: Database remains in valid state
  • Isolation: Concurrent transactions don't interfere
  • Durability: Committed changes are permanent

CAP Theorem

You can only guarantee 2 out of 3: Consistency, Availability, Partition tolerance

Caching Strategies

  • Cache-aside: Application manages cache
  • Write-through: Write to cache and database simultaneously
  • Write-behind: Write to cache first, database later
